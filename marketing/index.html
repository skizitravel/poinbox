How it works

1

Connect the shared inbox

An Admin connects Gmail or Outlook once. Operators work from the same inbound PO queue.

2

Extract and validate

PO headers and lines are parsed, matched against customer and item data, and checked for gaps.

3

Review exceptions

Low-confidence fields, product mismatches, and duplicate risks are routed into an operator queue.

4

Export clean orders

Reviewed orders can be exported as CSV now, with ERP destinations prepared for customer-specific rollout.

Built for controlled customer pilots

Mountain Goat keeps code and configuration separate from customer data. Runtime databases, OAuth tokens, API keys, and uploaded POs stay out of source control, and Admin-entered secrets are encrypted at rest.
